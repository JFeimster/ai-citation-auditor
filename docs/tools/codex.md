# Codex Instructions

## Use Codex for
- Adding new HTML pages
- Refactoring CSS
- Checking accessibility
- Fixing mobile layout issues
- Creating reusable page sections
- Updating README and documentation
- Creating comparison or SEO pages

## Do not use Codex to
- Invent proof, testimonials, customer logos, or performance metrics
- Add unnecessary frameworks
- Make financial, legal, compliance, or ranking guarantees
- Replace the static stack unless requested

## Good Codex task examples
- “Add a services.html page using the existing style system.”
- “Create a finance-content-checklist.html page and link it from the nav.”
- “Refactor repeated card styles into reusable CSS classes.”
- “Add JSON-LD SoftwareApplication schema to index.html without changing page copy.”
