# GitHub Copilot Guide

## Comment Patterns
- `// Keep this answer block short so AI can extract it.`
- `// Use fit-based wording here; do not promise outcomes.`
- `// Match FAQ schema to the visible FAQ exactly.`
- `// Link this hub to the supporting audit page.`

## Suggested Completions to Accept
- Shorter, clearer headings.
- Safer finance wording.
- Related links sections.
- Explicit CTA labels.
- Schema that matches visible content.

## Suggested Completions to Reject
- Any completion that adds guaranteed outcomes.
- Any completion that adds compliance approval language.
- Any completion that adds fake testimonials or stats.
- Any completion that inserts placeholder text.
- Any completion that creates duplicate thin pages.

## Guardrails for Finance Claims
- No guaranteed AI citations.
- No guaranteed rankings.
- No guaranteed traffic.
- No guaranteed approvals.
- No guaranteed funding.
- No guaranteed credit improvement.
- No financial, legal, tax, underwriting, lending, or compliance advice.

## CTA URL Preservation
- Preserve the main GPT URL exactly.
- Preserve mailto audit CTAs unless the page already has a better approved CTA.
- Do not swap in generic contact links.
