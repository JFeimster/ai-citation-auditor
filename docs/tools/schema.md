# Schema

## SoftwareApplication Schema
- Use for the GPT page or a tool page.
- Keep it grounded in visible content.
- Do not add pricing or offer fields unless the page truly shows them.

## FAQPage Schema
- Use only when the FAQ is visible on the same page.
- Match questions exactly.
- Match answers closely.

## BreadcrumbList Schema
- Use only when breadcrumbs are visible.
- Match the actual page hierarchy.
- Keep item positions correct.

## Organization Schema
- Use when the page describes the brand or site.
- Include real brand details only.
- Avoid padding with unsupported sameAs links.

## Article Schema
- Use only on article-style pages.
- Include real dates and real editorial fields.
- Do not use it on service pages or thin landing pages.

## When Not to Use HowTo Schema
- Do not use HowTo for pages without visible sequential instructions.
- Do not use HowTo for comparisons, service pages, or general guides without steps.

## Schema Validation Checklist
- Schema matches visible content.
- No hidden FAQ items.
- No fabricated offers or dates.
- No unsupported review or ranking data.
- Validator passes without critical errors.
