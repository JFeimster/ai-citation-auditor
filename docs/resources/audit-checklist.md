# Audit Checklist

## Content Checklist
- State the page purpose in the first screen.
- Answer the main question early and plainly.
- Use short sections with one idea per section.
- Prefer question-style H2s where they match reader intent.
- Include tables, lists, or FAQs when they make the page easier to scan.
- Remove filler, repetition, and vague opener paragraphs.
- Make the audience clear: finance operators, marketers, founders, editors, or compliance reviewers.

## SEO Metadata Checklist
- Unique title tag.
- Unique meta description.
- Absolute canonical URL.
- Absolute `og:url`.
- Matching `og:title` and `og:description`.
- Correct `og:type`.
- Correct `theme-color`.
- Favicon and shared asset paths loaded from `/assets/...`.

## AI Citation Readiness Checklist
- Direct answer block near the top.
- Clear entity names and topic labels.
- Enough context for AI to summarize safely.
- Short paragraphs and readable lists.
- Tables for comparisons or structured facts.
- Internal links to related support pages.
- Schema that matches visible content.

## Risky Claim Checklist
- Remove guaranteed approval language.
- Remove guaranteed funding or loan outcome language.
- Remove guaranteed credit improvement language.
- Remove guaranteed traffic, ranking, or citation language.
- Replace certainty with eligibility-based or fit-based wording.
- Add a review-before-publishing note where needed.
- Avoid implying legal, tax, underwriting, lending, credit, or compliance advice.

## Schema Checklist
- Only use schema that matches visible content.
- FAQPage schema only when the FAQ is visible on the page.
- Article schema only for article-style pages.
- BreadcrumbList only when breadcrumbs are visible.
- Organization schema only when it describes the site or brand.
- SoftwareApplication schema only for the GPT or tool pages.
- Do not use HowTo unless the page has real sequential steps.

## Internal Link Checklist
- Link to the core GPT CTA.
- Link to the relevant hub or pillar page.
- Link to one or two closely related support pages.
- Use descriptive anchor text.
- Avoid orphan pages.
- Avoid repeating the same anchor text everywhere.

## Final Pre-Publish Checklist
- No guaranteed outcome claims.
- No placeholder text.
- No broken internal links.
- No inline style blocks.
- No root `styles.css` or `script.js` references.
- One H1 only.
- Skip link present.
- Metadata complete.
- Schema matches the visible page.
- CTA points to the correct GPT or mailto target.
