# Skills

Practical skills:
- Inspect the repository before making changes.
- Preserve static-site conventions and finance-safe wording.
- Keep edits minimal when the request is scoped.
- Verify generated pages against the existing scaffold.

Boundaries:
- Do not invent guarantees, outcomes, or compliance claims.
- Do not add framework code or build tooling.
- Do not create root `styles.css` or `script.js`.

Review checklist:
- Single `h1`.
- Skip link present.
- Root-relative asset links.
- No inline styles.
