# Routing Logic Design

## Skill summary
Reusable process for routing logic design in the AI Citation Auditor repo.

## Inputs
- Source files or records
- Target goal
- Boundary notes
- Validation criteria

## Outputs
- A structured result
- A review checklist
- Safer next steps

## Step-by-step process
- Define the scope and source of truth.
- Inspect the input and the target shape.
- Apply the relevant audit, rewrite, or mapping rules.
- Check the result against safety and quality boundaries.
- Write the handoff or maintenance note.

## Good examples
- A clear, evidence-backed result.
- A process that stays within scope.

## Bad examples
- A vague, guarantee-heavy response.
- A result that loses traceability.

## Boundaries
- Do not claim approval, rankings, or outcomes.
- Do not hide unknown values.
- Do not expand scope without a reason.

## Review checklist
- Scope is explicit.
- Inputs are traceable.
- Output is reviewable.
- Boundaries are visible.
