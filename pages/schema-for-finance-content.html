Schema guide

Pick schema that matches the page, not schema that looks busy.

Schema helps when it describes visible content. On finance pages, the goal is fit, not decoration. The wrong schema can create confusion fast, especially when the page already deals with trust-sensitive claims.

Short answer

Use FAQPage for visible FAQs, Article for editorial pages, BreadcrumbList for real breadcrumbs, Organization for the brand, and SoftwareApplication for the GPT or tool page.

No HowTo schema unless the visible page truly contains sequential instructions.

Schema types

These are the common fit checks for finance content.

FAQPage

Use it only when the FAQ is visible on the same page and the questions match the JSON-LD exactly.

Article

Use it for article-style pages with a clear editorial structure, not for thin service pages or promos.

BreadcrumbList

Use it when the page visibly shows breadcrumbs that reflect the real page hierarchy.

Organization

Use it to describe the brand or site. Keep it grounded in real site details and logos.

SoftwareApplication

Use it for the GPT or other tool pages when the page is actually describing a software-like product.

HowTo

Do not use it unless the page has real sequential instructions that are visible to the reader.

Validation checklist

Check the schema against the HTML before you ship it.

  • Schema matches visible page content.
  • No FAQPage without a visible FAQ block.
  • No HowTo unless real steps are on the page.
  • No fabricated dates, offers, or review claims.
  • No schema that describes content the user cannot see.
  • Rich results validator passes without critical errors.

Common mistakes

Most schema problems come from trying to make the page look smarter than it is.

Mistake Why it is a problem Better move
FAQPage without a visible FAQSchema and page content no longer match.Add the FAQ visibly or remove the schema.
HowTo on a broad guideIt implies step-by-step instructions that do not exist.Use a normal article structure instead.
Article on a service pageIt can misrepresent the page type.Use Article only on actual editorial pages.
Organization with made-up detailsUnsupported schema is worse than no schema.Keep the brand data real and minimal.
